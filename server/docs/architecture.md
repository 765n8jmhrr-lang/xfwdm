# 拍版宝 后端架构设计文档

## 一、技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| **运行时** | Node.js 18+ | 高并发异步I/O，适合图片处理任务调度 |
| **Web框架** | Express.js | 轻量、生态成熟、中间件丰富 |
| **数据库** | MySQL 8.0 | 结构化数据存储，用户/订单/任务 |
| **缓存** | Redis 7 | 会话管理、任务队列、限流 |
| **对象存储** | 阿里云OSS / 腾讯云COS | 图片存储CDN加速 |
| **消息队列** | Redis Bull Queue | 异步任务调度，AI换装任务排队 |
| **AI推理** | Python + FastAPI | AI模型服务（IDM-VTON/OOTDiffusion） |
| **部署** | Docker + Docker Compose | 容器化一键部署 |
| **反向代理** | Nginx | SSL终止、负载均衡、静态资源 |

## 三、核心模块划分

### 1. 用户认证模块 (auth)
- 微信小程序登录（code → openid/session_key）
- JWT Token 签发与刷新
- 用户注册/信息更新

### 2. 图片上传模块 (upload)
- 图片格式/大小校验
- 压缩处理（Thumbnails）
- 上传至对象存储（OSS/COS）
- 返回图片URL

### 3. AI换装任务模块 (tryon)
- 创建换装任务（入队列）
- 任务状态查询（pending/processing/completed/failed）
- 结果获取
- 超时/重试机制

### 4. 用户额度与支付模块 (payment)
- 额度查询与扣减
- 微信支付接口对接
- 套餐管理
- 支付回调处理

### 5. 历史记录模块 (history)
- 任务结果列表查询（分页）
- 单条记录详情
- 删除记录（软删除）

### 6. AI Worker 服务 (Python)
- 独立的 Python FastAPI 服务
- 消费 Redis 队列中的换装任务
- 加载 AI 模型进行推理
- 结果图上传至 OSS
- 通过回调/状态更新通知主服务

## 四、数据流程

```
用户操作流程：
1. 用户打开小程序 → wx.login() → 后端验证 → 返回JWT Token
2. 用户上传服装图 → POST /api/upload → 返回 garment_url
3. 用户上传模特图 → POST /api/upload → 返回 model_url
4. 用户点击"开始换装" → POST /api/tryon/create
   → 检查额度 → 扣减额度 → 创建任务 → 入队列 → 返回 task_id
5. 前端轮询 → GET /api/tryon/status?task_id=xxx
   → 返回状态（processing/completed）
6. AI Worker 消费队列 → 下载图片 → 推理 → 上传结果 → 更新任务状态
7. 前端获取结果 → GET /api/tryon/result?task_id=xxx
   → 返回 result_image_url
```

## 五、安全设计

- **接口鉴权**：JWT Token + 中间件拦截
- **请求限流**：Redis 令牌桶，单用户 10次/分钟
- **图片安全**：上传时触发内容审核（腾讯云/阿里云图片审核API）
- **数据加密**：HTTPS全链路加密，敏感数据AES加密存储
- **SQL注入**：ORM参数化查询
- **XSS/CSRF**：输入过滤、Token校验

## 六、性能优化

- **CDN加速**：图片通过CDN分发，全国用户就近访问
- **Redis缓存**：热点数据缓存（用户信息、额度）
- **队列削峰**：AI任务异步排队，避免瞬时高并发
- **GPU调度**：多Worker并行处理，动态扩容
- **数据库连接池**：复用连接，降低开销
- **图片压缩**：上传时自动压缩至合适尺寸
